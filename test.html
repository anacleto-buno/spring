SE Test Task: Dynamic Product Catalog Filter

 

This task evaluates your ability to design and implement robust APIs, manage data effectively on the backend, and develop efficient, scalable search functionalities. A basic user interface will serve to demonstrate the functionality and interaction with your backend solution.

 

Goal: Create a simple web or mobile application that allows users to dynamically filter a list of product items based on their input.

 

Setup:

  • Backend: Choose a backend framework/language (e.g., Node.js with Express, Python with Flask/Django, Ruby on Rails, Go etc.).

  • Database: A simple database solution (e.g., SQLite, PostgreSQL, MongoDB, MySQL).

  • Frontend: JavaScript with any UI framework; Flutter or React Native/Expo if already familiar. 

Core Requirements:

  1. Backend API for Data Generation & Storage:

    1. Data Generation Endpoint: Create an API endpoint (e.g., POST /products/generate) that, when called, generates at least 1000 "product" records and stores them in the chosen database. The product properties remain the same: name, description, category, brand, price, stock quantity, SKU, release date, availability status, customer rating, available colors and sizes. This endpoint could optionally take parameters for the number of products to generate or specific seed data.

    2. Product Retrieval Endpoint: Create an API endpoint (e.g., GET /products) that returns a list of product records from the database.

  2. Backend API for Dynamic Filtering:

    1. Search Endpoint: Create an API endpoint that accepts a search query.

    2. Backend Filtering Logic: The backend should implement the core filtering logic. It will query the database for products where any of its properties (name, description, category, brand, or SKU) contain the typed search term.

  3. Frontend Integration:

    1. Initial Data Load: The frontend application will call the GET /products on initial load to display the first set of products.

    2. Search Input: Provide a single input field (a text box) where the user can type their search query.

    3. Dynamic Filtering with API Calls: As the user types, the frontend should make API calls to the GET /products/search endpoint.

    4. Display Results: Display the filtered list of products in a clear and readable format.

  4. Performance & Responsiveness:

    1. The filtering process should feel fast and smooth, even with 1000+ items and multiple characters typed.

  5. Host the solution, or provide clear and reproducible steps to run the project.

  6. Testing:

    1. Implement automated tests for areas deemed critical for testing.

  7. Assumptions, tradeoffs and improvements: 

    1. Please write down any assumptions you made during development, any tradeoffs you chose to make (e.g., sacrificing performance for the sake of simplicity), and any improvements you would consider if you had more time.

  8. Your Own Ideas: Feel free to implement any additional features or improvements you believe would enhance the application or demonstrate your skills.
